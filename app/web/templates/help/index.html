1. Getting Started

The Printer Dashboard watches your networked printers in real time, tracks toner and drum supply levels, records page counts, and alerts you (via email) when something needs attention. Everything you need is available from the navigation bar at the top of every page.

Top navigation

  • Dashboard — live overview of every active printer
  • Discover — scan a subnet to find printers to add
  • Alerts — chronological log of every alert ever raised
  • Reports — six analytical reports with CSV export
    • {% if current_user.is_admin %}
  • Config Admin — all settings live here
    • {% endif %}

Your account menu

Click your username in the top-right corner to open the account menu. From there you can Change Password, open this Help page, or Log out.

Tip: If an admin creates your account with an email address, you'll receive a welcome email with a temporary password. The first time you log in, you'll be required to set a new password before you can access any other page.

2. Dashboard Overview

The Dashboard shows every active printer as a tile. Each tile summarizes the printer's current status and its lowest supply level.

Status dots

  • Green — printer is online and responding to SNMP
  • Red — printer is offline (missed several polls)
  • Gray — not yet polled or status unknown

Supply bars

  • Green — above the warning threshold (healthy)
  • Orange — below warning threshold
  • Red — below critical threshold

Filter bar

Above the tiles, use the filter pills to narrow down by location, status, or search by name. Filters stack — picking a location and "online only" shows only online printers at that location.

3. Managing Printers

Adding a printer {% if not current_user.is_admin %}Admin{% endif %}

Admin users click a printer tile → the printer detail page opens. To add a new one, use Discover to scan a subnet OR click "Add Printer" from the printer list page. You provide the IP address, optionally a display name, and the SNMP community string (usually public). The first poll runs automatically.

The printer detail page

Everything you need is on one page:

  • Asset info — display name, IP, vendor, model, serial, assigned person, SQL number, phone ext., web login credentials
  • Supply Levels — live bars for every toner, drum, or ink cartridge
  • Toner Replacement History — every time a cartridge was swapped, with an admin-editable cost field
  • History link — opens the per-printer History page (next section)
  • Recent alerts — the last dozen alerts raised for this printer

The History page

Click the History button on a printer's detail page to open a deeper view with depletion estimates and historical charts.

  • Date range pills at the top right — switch between 7 days / 30 days / 90 days / All time (default: 90 days)
  • Active Supplies table — for every cartridge: color, status (OK / Warning / Critical), current %, consumption rate (% per day), days remaining at the current rate, predicted empty date
  • Pages-per-day card — average page output over the selected window
  • Supply Levels Over Time chart — one line per supply, with vertical dashed markers showing each replacement event (color-coded to the cartridge)
  • Page Count Over Time chart

Depletion estimates are replacement-aware: the regression line is fitted only to readings after the most recent toner_replaced / drum_replaced event, so days-remaining stays accurate right after you swap a cartridge.

Actions available on the detail page

  • Poll Now — force an immediate SNMP poll instead of waiting for the scheduler
  • Resend Alerts — clear dedup flags so active warnings are re-emailed on the next poll (admin only)
  • Create Helpdesk Ticket — email a ticket to the configured helpdesk address
  • Set Thresholds — override the site-wide warning/critical percentages for this one printer (admin only)
  • Remove Printer — soft-delete. Printer data is kept; printer can be restored from Config → Removed Printers (admin only)

4. Supply Levels & Alerts

How supply levels are read

Every poll, the dashboard queries each printer's SNMP supply table. Level is reported as a percentage of capacity. Some older printers only report a raw count vs. max — the dashboard converts that to a percentage automatically.

Warning vs. Critical thresholds

Default thresholds are 15% (warning) and 5% (critical). When a supply crosses below warning for the first time, a warning email is sent. When it later crosses critical, a critical email is sent. Each email is sent once per supply lifecycle — a new cartridge install resets the flags so the cycle starts over.

Toner replacement detection

When the supply level jumps upward significantly (e.g. 3% → 98%), the system records a toner_replaced event. You can enter the cost of the replacement from the printer detail page so it flows into the Toner Cost report.

Offline alerts

If a printer fails to respond to the scheduled poll several times in a row (default: 3 consecutive misses), it's marked offline and an offline alert email is sent. When it responds again, a back-online email is sent.

Emails only send if SMTP is configured. If you're not getting alert emails, start with Config → Email / SMTP and send a test email from that page.

5. Reports

The Reports tab (top nav) has six pre-built reports. All are visible to every signed-in user. Each report has a filter bar (date range, printer, etc.), a chart, a sortable table, and a Download CSV button for exporting the underlying data.

Print Volume

Pages printed per printer in a date range. Group by printer, person, or location.

Page Count Over Time

Multi-line chart of cumulative page count per printer over time.

Toner Cost

Every toner replacement with the cost you entered. Totals by printer and color.

Cost Per Page

Total toner cost ÷ total pages printed. The printer efficiency metric.

Consumption Rate

How fast each supply depletes (% per day), with projected days remaining.

Reliability

Offline event counts per printer. Higher numbers = more unreliable printer.

Entering replacement costs

Admin: open a printer detail page, scroll to Toner Replacement History, and fill in the Cost field for a replacement row. The cost immediately flows into the Toner Cost and Cost Per Page reports.

6. Remote Agents Admin

A Remote Agent is a small background service running on a machine at another site. It scans the local subnet for printers and reports back to this central dashboard over HTTPS — no VPN or port-forwarding required. Use agents when you need to monitor printers on networks the dashboard server can't reach directly.

Adding an agent

  1. Go to Config → Remote Agents.
  2. Enter a name (a Location with the same name is auto-created).
  3. Optionally enter a subnet — leave blank to have the agent auto-detect its own.
  4. Click Create Agent.
  5. Copy the Windows PowerShell or Raspberry Pi bash one-liner displayed.
  6. Paste it into an elevated PowerShell (Windows) or shell (Pi) on the remote machine.

Managing agents

Each agent row in the table has action buttons:

  • Rescan Now — tell the agent to scan immediately
  • Force Update — push the latest agent script
  • Edit Interval — change scan frequency
  • Edit Subnet — push a new subnet (blank = auto-detect)
  • Assign Location
  • Regenerate API Key
  • Delete — queues uninstall on next check-in

Auto-update

When the dashboard and an agent fall out of version sync, the dashboard auto-queues an update command on the next check-in. The agent downloads the new script and restarts itself via its scheduled task / systemd service. An Outdated badge appears in the Version column, and a Update All Outdated button in the agents card header lets you push updates to multiple agents at once.

Manually triggering a check-in

On the remote machine:

# Windows
Start-ScheduledTask -TaskName "PrinterAgent"

# Linux / Raspberry Pi
sudo systemctl restart printer-agent

7. User Accounts & Security

Roles

  • Viewer — read-only. Can see all printers, reports, and alerts. Cannot edit anything.
  • Admin — full access. Can add/edit/remove printers, manage users, change settings, apply updates.

Temporary passwords & forced change

When an admin creates a new user (or resets an existing user's password via Send Reset Email), the system generates a random temporary password. On first login with that password, the user is redirected to a Set Your Password page and cannot access any other page until they choose a new password (minimum 8 characters).

Changing your own password

Click your username in the top-right → Change Password. You'll need to enter your current password to confirm.

Resetting another user's password Admin

Go to Config → Users. For each user you have two options:

  • Send Reset Email — generates a new temp password, emails it to the user, forces change on next login. Requires an email address on file.
  • Manual set — type a password and click Set. Use this if the user has no email address or you need immediate offline access.

8. Admin Configuration Admin

Everything below lives under Config (the gear icon in the top nav).

Email / SMTP

Configure the outgoing mail server. Three auth modes are supported: STARTTLS (port 587 — most common), SSL/TLS (port 465), and None (local relay). Send a test email from this tab to verify.

Alert-To is where all alert emails go. Helpdesk email is where manually-created tickets (from a printer's detail page) are sent.

Thresholds

Set the site-wide warning and critical percentages for supply level alerts. Any printer can override these from its detail page (the override goes onto that printer's row in the database).

Reset Per-Printer Overrides — the same tab has a one-click button that bulk-clears every per-printer override at once. Useful when you change the site default and want all printers to inherit the new value (the site default doesn't automatically apply to printers with their own overrides). The button shows the count of printers currently overridden and is disabled when there's nothing to clear.

Predictive Toner Alerts

When enabled, a scheduled job runs hourly and uses linear regression on recent supply-level readings to predict when each cartridge will run out. If predicted to be empty within N days (default 7), a helpdesk ticket is auto-generated — once per supply lifecycle. Set the days threshold and minimum data-point count from the Alert Settings tab.

Locations

Tag each printer with a location. Locations power the filter bar on the dashboard and the grouping options in reports.

Import

Upload an .xlsx spreadsheet to pre-populate printer asset fields (assigned person, SQL number, location, etc.). Rows match printers by IP. Pending records are applied to existing printers or auto-filled when a new printer with a matching IP is added.

Software Updates

The Updates tab compares the running version against the latest GitHub release. When a newer version is available, release notes are shown along with a copyable block of commands to run.

Why updates are applied manually: the dashboard intentionally does not have permission to modify its own host or containers. This keeps the attack surface very small — even in the unlikely event the web app is ever compromised, the attacker cannot use the app to pivot to the VM. To update, SSH into the server, pull the latest code, and rebuild the container. The whole process takes about a minute.

On the VM:

cd /path/to/network-printer-dashboard
git pull
docker compose up --build -d

Or save yourself the typing next time by creating an update.sh script in the repo with those three lines and making it executable (chmod +x update.sh).

Activity Log

Every administrative action is recorded with timestamp, username, action type, target, and detail. Retained for 30 days. Exportable as CSV.

Backup / Reset

Download a SQL dump of the full database, or reset the instance to factory defaults.

9. Troubleshooting / FAQ

I'm not getting alert emails

  1. Check Config → Email / SMTP and send a test email.
  2. Verify the Alert-To address is set.
  3. Check the individual alert toggles under Config → Alert Settings.
  4. If using Gmail, make sure you're using an App Password, not your regular password.

A printer shows as "offline" but I know it's online

  1. Check that SNMP is enabled on the printer (some vendors ship it disabled).
  2. Verify the community string matches (usually public).
  3. Make sure the dashboard server can reach the printer IP (no firewall blocking UDP/161).
  4. Click Poll Now on the printer detail page to try again.
{% if current_user.is_admin %}

A remote agent shows "Outdated" but never updates

Log into the agent machine and trigger a check-in manually (see section 6 above). The agent will receive the queued update command on the next successful check-in.

An update failed — how do I roll back?

Since updates are applied manually via git pull, rolling back is just as easy: 

cd /path/to/network-printer-dashboard
git log --oneline -5              # find the previous good commit SHA
git reset --hard <previous-sha>
docker compose up --build -d

I get "SMTP not configured" but I already filled in settings

Check that all four fields (host, port, user, password) are filled and the auth mode is set correctly. If you set auth to "None", only the host field is required. If you changed settings in the UI, the new values take effect immediately — no restart required.

{% endif %}

How often are printers polled?

Default is every 60 minutes. {% if current_user.is_admin %}Change it at Config → Thresholds → Poll interval.{% endif %} Remote-agent printers are polled on the agent's own interval.

How long is data kept?

Telemetry and supply snapshots are kept indefinitely (small rows, small cost). Alerts are kept indefinitely too. The activity log auto-prunes at 30 days.
